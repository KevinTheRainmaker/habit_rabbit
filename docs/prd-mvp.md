## 1. Executive Summary

Habit Rabbit은 토끼굴 세계관과 당근 포인트 경제를 기반으로 한 프리미엄 습관 트래커 앱이다. 습관 앱 포화 시장에서 감성적 차별화와 심리적 안전망(스트릭 복구권)으로 리텐션을 확보하고, AI 코칭과 실패 패턴 분석을 핵심 유료 가치로 수익화한다. MVP는 가족/팀 스페이스와 현금 IAP를 제외한 개인 사용자 대상 프리미엄 모델로 출시한다.

## 2. Background & Context

### 시장 문제

Reddit 커뮤니티(r/getdisciplined, r/Habits 외 4개) 리서치에서 확인된 습관 앱의 핵심 실패 패턴:

1. **메타습관 실패**: 다운로드 → 며칠 사용 → 망각 → 삭제 사이클 반복
2. **체크박스 역전**: 트래킹 자체가 목적이 되어 실제 습관 형성 방해
3. **스트릭 스트레스**: 하루 실수 → 죄책감 → 전체 포기 (올-오어-낫씽)
4. **앱 포화 피로감**: "또 다른 앱"에 대한 근본적 불신
5. **가치 전 업셀**: 가치 체감 전 결제 요청에 강한 반감
6. **점진적 설계 부재**: 처음부터 큰 목표 → 초기 실패 → 이탈

### 기회

- 기존 앱들은 스트릭 실패 시 이탈을 방치함 → **복구권**이 직접 해결
- 감성/캐릭터 기반 앱이 없음 → **토끼 세계관**이 시각적 차별화
- AI 코칭 제공 앱이 드물고 품질이 낮음 → **실패 패턴 기반 맞춤 피드백** 차별화
- 게이미피케이션 + 수집 경제를 결합한 앱 없음 → **당근 포인트 + 아이템 꾸미기**

## 3. Objectives & Success Metrics

### Goals

1. 출시 6개월 내 Day-30 리텐션 35% 달성
2. 출시 6개월 내 무료→유료 전환율 3% 달성
3. 스트릭 첫 실패 후 D7 이탈율을 50% 이하로 유지

### Non-Goals (명시적 제외)

| 제외 항목                        | 이유                                            |
| -------------------------------- | ----------------------------------------------- |
| 가족/팀 스페이스                 | MVP 범위 초과. 개인 검증 후 다음 단계           |
| 현금 IAP (당근 팩)               | 결제 시스템 복잡도 증가. 구독 수익 검증 후 추가 |
| HealthKit / Google Fit 자동 연동 | 엔지니어링 비용 대비 MVP 우선순위 낮음          |
| 스마트 알림 타이밍 학습 (ML)     | 데이터 축적 필요. v2 이후 적용                  |
| 소셜 피드 / 랭킹                 | 커뮤니티 관리 부담. 핵심 루프 검증 후 검토      |

### Success Metrics

| 지표                 | 현재     | 목표     | 측정 방법   |
| -------------------- | -------- | -------- | ----------- |
| Day-30 리텐션        | - (신규) | 35%      | 코호트 분석 |
| Day-7 리텐션         | - (신규) | 55%      | 코호트 분석 |
| 무료→유료 전환율     | - (신규) | 3%       | RevenueCat  |
| 첫 실패 후 D7 이탈율 | - (신규) | 50% 이하 | 이벤트 로그 |
| 온보딩 완료율        | - (신규) | 70%      | 퍼널 분석   |
| 일일 습관 완료율     | - (신규) | 60%      | 앱 내 로그  |

## 6. Solution Overview

### 핵심 루프

```
습관 완료 → 당근 포인트 적립 (스트릭 보너스) → 아이템 구매 → 토끼/굴 꾸미기
     ↑                                                              ↓
     └────────────── 애착 형성 → 다음날 앱 열기 ──────────────────┘
```

### 프리미엄 모델 구조

| 구분    | 무료                    | 유료                           |
| ------- | ----------------------- | ------------------------------ |
| 습관 수 | 최대 3개                | 무제한                         |
| 알림    | 고정 시간 1개           | 요일/시간 세분화               |
| 통계    | 기본 (스트릭, 달성률)   | 심화 (패턴, 추세, 실패 분석)   |
| AI 코칭 | -                       | 실패 패턴 분석 + 맞춤 피드백   |
| 복구권  | 최초 1회 무료 체험      | 월 1개 자동 지급 (적립 가능)   |
| 아이템  | 기본 아이템 + 미션 보상 | 프리미엄 아이템 추가 구매 가능 |

### 업셀 플로우 (3단계)

1. **자연 업셀** (S3-c): 4번째 습관 추가 시도 → 유료 제안
2. **감정 업셀** (S3-a): 첫 스트릭 실패 → 복구권 무료 체험 → 유료 전환
3. **가치 노출** (S3-b): 심화 통계/AI 코칭 블러 티저 → 지속 노출

**원칙**: 최소 3일 사용 후 업셀. 감정적 순간에만 직접 전환 요청.

### 기술 스택 (결정 예정)

- **구독 관리**: RevenueCat (A/B 테스트, 복원, 영수증 검증 포함)
- **개발 형태**: 1인 개발, 크로스플랫폼 (iOS/Android)
- **백엔드**: 최소화 (로컬 퍼스트 + 클라우드 싱크)

### 디자인 원칙

- 토끼굴 세계관 일관성: 모든 UX 언어를 캐릭터 관점으로 작성
- "체크박스"가 아닌 "습관 형성"을 강조하는 카피
- 실패 시 처벌 없는 언어: "0일 리셋" 대신 "오늘 다시 시작"

## 8. Timeline & Phasing

### Phase 0: 수요 검증 (개발 전, 2주)

- 랜딩 페이지 제작 (토끼굴 꾸미기 비주얼 중심)
- 사전 등록 캠페인 → 목표: 200명
- **Go 기준**: 200명 달성 → Phase 1 진행. 미달 → USP 재검토

### Phase 1: MVP 개발 (P0 전체, P1 일부)
